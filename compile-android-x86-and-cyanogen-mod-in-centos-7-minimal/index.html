解决64位CentOS 7 Minimal环境下的安卓编译问题

发表于 2016-01-13   |   分类于 安卓

前段时间攒了个下载机，装的是64位 CentOS 7（CentOS Linux release 7.2.1511），Minimal最小化安装。把以前老机器上的 Android-x86 源码和 CyanogenMod Android 的源码拷了份过来，配置一下编译环境，在此处记录一下。

1 . 编译 android x86

首先按照android官网编译环境的配置，安装一些需要的工具：

注意官方给出的是ubuntu的工具名，我们这里安装64位Centos对应的工具。

yum install flex.x86_64
yum install bison.x86_64
yum install gperf.x86_64

接下来，说一下JDK的问题。我系统用的是 Oracle JDK 8，版本太高，编译过程中提示只能用 JDK6或者7。找了个JDK 7，发现还是报 “error: ordinal has private access in Enum” 错误，最后换成JDK 6，就可以顺利通过。

编译过程使用与系统环境不同的JDK配置，编译启动脚本如下：
`

#!/usr/bin/env bash
ANDROID_HOME=~/jdks/jdk1.6.0_45
PATH=$ANDROID_HOME/bin:$PATH
nohup make -j4 iso_img TARGET_PRODUCT=android_x86 > build.log 2>&1 &
`
下面针对编译过程中出现的各种问题错误，依次解决如下：

  • 错误：prebuilts/misc/linux-x86/ccache/ccache: /lib/ld-linux.so.2: bad ELF interpreter: No such file or directory
    解决：yum install glibc.i686 （64位环境中的32位库）

  • 错误：bc: command not found
    解决：yum install bc.x86_64

  • 错误：error while loading shared libraries: libgcc_s.so.1: cannot open shared object file: No such file or directory
    解决：yum install libgcc.i686 （64位环境中的32位库）

  • 错误：error while loading shared libraries: libz.so.1: cannot open shared object file: No such file or directory
    解决：yum install libzip.i686 （64位环境中的32位库）

  • 错误：error while loading shared libraries: libstdc++.so.6: cannot open shared object file: No such file or directory
    解决：yum install libstdc++.i686 （64位环境中的32位库，这个依赖libgcc.i686，所以装这个就会装上libgcc.i686）

  • 错误：ImportError: No module named mako.template
    解决：yum install python-mako.noarch

  • 错误：patch: command not found
    解决：yum install patch.x86_64

  • 错误：bash: mkisofs: command not found
    解决：yum install genisoimage.x86_64
    至此，Android X86已经可以编译通过，生成可安装的 Android x86 ISO。

2 . 编译 CyanogenMod Android

在上面成功编译完 Android x86 配置的基础上，编译 CyanogenMod 的时候，会遇到的问题及解决：

  • 错误：build/envsetup.sh: line 1934: schedtool: command not found 或者 bash: schedtool: command not found
    解决：原本想的和上面一样 yum install schedtool轻松解决问题的，结果告诉我 Warning: No matches found for: schedtool，schedtool在yum库里找不到，只好去找源码安装。
    代码下载页：http://slackbuilds.org/repository/13.37/system/schedtool/
    直接下载地址：http://freequaos.host.sk/schedtool/schedtool-1.3.0.tar.bz2
    校验码 (0d968f05d3ad7675f1f33ef1f6d0a3fb)
    解压缩，tar -xjf schedtool-1.3.0.tar.bz2
    （如果像我一样，不默认装进系统目录，而只是安装进开发者账号目录里，则打开Makefile，修改 DESTPREFIX=/home/xxx/developlocal）
    然后执行 make & make install，完成安装。
    然后按照正常流程，执行breakfast mako ，接着 brunch mako，就顺利完成编译，得到对应的系统镜像文件了。
